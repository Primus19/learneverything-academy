LearnEverything Academy - Amplify Deployment Instructions

This document provides instructions for deploying the LearnEverything Academy static website to AWS Amplify.

Note: This is a pure static HTML site with no build dependencies, which ensures it will deploy successfully on Amplify without any build process issues.

Deployment Steps

  1. Log in to the AWS Management Console

    Navigate to the AWS Amplify service.

  2. Create a new app

    Click "Create app" and select "Deploy without Git provider" option.

  3. Upload the zip file

    On the "Deploy without Git provider" screen, provide an app name (e.g., "LearnEverythingAcademy") and upload the provided zip file containing the static website.

  4. Save and deploy

    Click "Save and deploy" to start the deployment process.

  5. Wait for deployment to complete

    Amplify will deploy your static site. This should complete successfully since there is no build process required.

  6. Access your website

    Once deployment is complete, Amplify will provide a URL where your site is accessible (typically something like https://main.xxxxxxxxxxxx.amplifyapp.com).

Why This Approach Works

The previous deployment attempts failed because they were using a Next.js application with TypeScript, which required a complex build process that was encountering errors in the Amplify environment.

This solution uses a pure static HTML approach with the following advantages:

  • No build process required - The site consists of plain HTML, CSS, and JavaScript files that can be served directly without any compilation or build steps.
  • No dependencies - The site uses Tailwind CSS via CDN, eliminating the need for npm packages or other dependencies.
  • Simplified deployment - The included amplify.yml file explicitly tells Amplify that no build process is needed.

Troubleshooting

If you encounter any issues during deployment:

  1. Verify the amplify.yml file

    Ensure the amplify.yml file is at the root of your zip file and has the correct configuration.

  2. Check file permissions

    Make sure all files have appropriate read permissions.

  3. Review Amplify logs

    If there are any issues, check the build logs in the Amplify console for specific error messages.

Future Updates

To update the site in the future:

  1. Modify the HTML, CSS, or JavaScript files as needed
  2. Create a new zip file containing all the updated files
  3. In the Amplify console, select your app and click "Manual deploy"
  4. Upload the new zip file and deploy
Important: If you need to add more complex functionality in the future, consider setting up a proper CI/CD pipeline with a Git provider for more streamlined updates.
